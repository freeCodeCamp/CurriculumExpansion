Main Ideas for Basics in Testing

  1. Typical development cycle: Code something -> Run the code -> Verify that the code functions as expected.
  2. For every change that you make, you have to verify every single function every single time.
  3. Manually testing code can be time consuming.
  4. The testing process can be automated with assert libraries and testing frameworks outside of your main code.
  5. Assert libraries check to see if the result of your code is equal to the value that you expect it to be.
  6. Testing frameworks help you to define and run the tests that you write for your code.
  7. In this code sandbox, we have a Garden program in the "src" directory that you can write tests against. The requirements are below. Tests ran can be verified in the "Tests " tab.

Garden Program Requirements

Garden

  • The program can make a garden
  • The garden can plant different kinds of plants
  • The garden can only hold at most 5 fruit trees, 10 vegetable plants, and 7 miscellaneous plants

Plants

  • Plants can be given a name
  • The plants need to be watered
  • If the plants are neglected (not watered), the plants will wither after three days
  • The plants need to be watered to recover.

Fruit Trees/Vegetable Plants

  • Fruit trees and Vegetable plants have the same requirements as plants
  • Fruits and vegetables to fruit trees and vegetable plants respectively
  • When you harvest fruits or vegetables, you get a count of how many fruits or vegetables harvested
